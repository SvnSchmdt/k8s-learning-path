# Exercises

Practice challenges to reinforce what you've learned in the modules and labs. Work through these after completing the corresponding phases of the learning path.

---

## Exercise sets

| Set | Level | Topics covered |
|-----|-------|----------------|
| [Beginner](beginner.md) | Beginner | Pods, Deployments, Services, ConfigMaps — core kubectl skills |
| [Intermediate](intermediate.md) | Intermediate | Ingress, RBAC, Storage, Helm, Kustomize |
| [Troubleshooting](troubleshooting.md) | All levels | Debugging broken clusters, failed pods, misconfigured resources |

---

## How to use exercises

Each exercise follows this pattern:

1. **Scenario** — A short description of the situation
2. **Task** — What you need to accomplish
3. **Hints** — Optional pointers if you get stuck
4. **Solution** — A collapsible section with one possible solution

> Try to solve each exercise without looking at the solution first. It is fine to check the hints.

---

## Recommended sequence

- After Phase 2–3 (Modules 02–04): **[Beginner exercises](beginner.md)**
- After Phase 4–5 (Modules 05–10): **[Intermediate exercises](intermediate.md)**
- Any time: **[Troubleshooting exercises](troubleshooting.md)** — these build the debugging instincts you need for production and for CKA
